# Как исправить проблему с политикой выполнения PowerShell

## Проблема
При попытке активировать виртуальное окружение Python возникает ошибка:
```
выполнение сценариев отключено в этой системе
```

## Решение 1: Изменить политику выполнения (Рекомендуется)

Запустите PowerShell **от имени администратора** и выполните:

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser -Force
```

Или используйте готовый скрипт:
```powershell
.\fix-powershell-execution-policy.ps1
```

После этого вы сможете активировать виртуальное окружение:
```powershell
.\venv\Scripts\activate
```

## Решение 2: Использовать альтернативный метод активации

Вы можете использовать файл `activate-venv.bat`, который не требует PowerShell:
```cmd
activate-venv.bat
```

Или напрямую через Command Prompt (cmd.exe):
```cmd
venv\Scripts\activate.bat
```

## Решение 3: Обход политики для одной сессии

Если вы не хотите менять политику выполнения, можете обойти её для текущей сессии:

```powershell
powershell -ExecutionPolicy Bypass -File venv\Scripts\Activate.ps1
```

## Что означает RemoteSigned?
- **RemoteSigned** - разрешает выполнение локальных скриптов (подписанных или нет)
- Безопасно для разработки
- Требует подписи только для скриптов, загруженных из интернета

## Альтернативные политики
- `Bypass` - отключает все ограничения (менее безопасно)
- `Unrestricted` - разрешает все, но показывает предупреждения
- `RemoteSigned` - рекомендуется для разработки (текущая настройка)
